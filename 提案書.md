# Shopify構築・運用タスク 役割分担および要件定義書

本ドキュメントは、アパレルセレクトショップのShopifyストア構築における、担当者（玉記）とクライアント間の役割分担、および実運用に向けた要件定義をまとめたものです。

---

## 1. 玉記の実行タスク（Shopify構築・運用保守）
実際に手を動かしてShopify上のシステムを構築し、テストや保守を行う「開発・システム管理」の領域です。設定（支払い等）は範囲外とします。

### 構築作業（フェーズ2〜4）
* **テストデータとカテゴリの仕込み**
  * ダミー画像、仮SKU、バリエーションを用いて最低5〜8点のテスト商品を登録する。
  * `ALL ITEMS`、`NEW ARRIVALS`、`TOPS`、`PANTS`などの自動コレクション（カテゴリ）の箱を作成する。
* **テーマのカスタマイズ（フロントエンド構築）**
  * 背景色（`#F5F5F5`）、テキスト色（`#333333`）、フォント（`Helvetica`）を設定する。
  * グリッドの1px罫線やコンテナの角丸リセット等、カスタムCSSを全体に適用する。
  * **トップページ**: ファーストビュー、ブランドリスト（ホバーで画像が浮かび上がるカスタムLiquid）、新着一覧、アシンメトリー画像セクションを配置する。
  * **商品ページ**: 画像ギャラリーの比率（3:4等）設定、サイズ表（Size Guide）モーダル、下部に関連商品（クロスセル）セクションを配置する。
  * **コレクションページ**: 絞り込み（ファセットフィルタ）の有効化、商品グリッドの列数設定（PC: 3〜4列、スマホ: 2列）を行う。
* **固定ページの作成**
  * `About Us`、`Contact`（問い合わせフォーム）、`FAQ`（アコーディオン）、`Size Guide`ページの枠組みとレイアウトを構築する。

### システム設定・テスト（フェーズ1、5）
* 構築中のテストモードの「ON/OFF」設定を行う。
* 税金設定（「すべての料金は税込みです」のチェック）を行う。
* 注文確認・発送完了等の通知メールテンプレートのプレビュー確認とロゴ挿入調整を行う。
* 疑似決済（テスト注文）、発送・返金テスト、モバイルUXの徹底確認を行う。
* 実在庫到着後、パスワード保護を解除しサイトを一般公開する。

## 2. 玉記からの「提案」タスク
システムを破綻させないためのデータルールや、ECサイトとしての最適な運用方針を提案し、クライアントと合意形成を行う領域です。

* **SKU（品番）ルールの策定と提案**
  * 将来的なCSV一括処理やAPI連携時にエラーを防ぐため、厳密なフォーマットを提案する。
  * 【推奨フォーマット例】`[ブランド3桁]-[シーズン2桁]-[カテゴリ2桁]-[カラー2桁]-[サイズ2桁]`
  * 【適用例】`UNIQLO / 2026SS / ロングスリーブTシャツ / 黒 / L` → `UNI-26S-LT-BK-LO`
* **商品マスタの管理手法の提案**
  * 初期は「スプレッドシート」でのマスタ管理を提案する。
  * 将来的な拡張として「SQLite (RDBMS) での内製ビューアプリ」等も視野に入れるが、管理数が少ないうちは無駄な課金ツールを避ける方針をすり合わせる。
* **タグ付け（Tagging）ルールの策定**
  * 表記揺れを防ぐためのタグマスタ（例: `メンズ`, `レディース`, `アウター`, `26AW`, `定番`）を作成し提案する。
* **デザインガイドラインの合意**
  * 提案済みのカラーパレットやフォントについての最終確認を行う。
* **独自ドメインの取得方針**
  * 自社ドメインを購入するかどうかの検討事項としてクライアントに提示する。
* **配送・送料ルールの構築**
  * 基本配送料（例：全国一律800円、15000円以上送料無料）や、ネコポス/宅急便の切り分け（配送プロファイル）方針を提案する。
* **法的ポリシーのひな形提供**
  * AIで出力した特商法、プライバシーポリシー、返金ポリシー、利用規約のベースを提供し、内容の確認と修正を促す。

## 3. クライアントが「実行・用意」するタスク
契約、支払い、自社情報の登録など、玉記が代行できない（すべきではない）運営・事業領域です。

* **アカウント・インフラ関連**
  * Shopifyのベーシックプランの契約と、月額料金用のクレジットカード情報の設定。
  * （提案を受けて）独自ドメインを購入する場合の契約・更新手続き。
* **決済プロバイダーの登録・審査**
  * Shopify Paymentsの有効化、売上金入金用の銀行口座の登録。
  * Shop Pay、Apple Pay、Google Payなどの有効化に伴う事業情報の入力。
* **自社情報（法的表記）の提供**
  * 特定商取引法に基づく表記に記載する事業者の正式名称、住所、電話番号の確定と入力。
  * 提供された法的ポリシーひな形（お客様都合の返金不可条件など）の最終チェックと合意。
* **クリエイティブ・原稿素材の用意**
  * サイトロゴ、およびファビコン（16x16px / 32x32px）の画像データの提供。
  * 各固定ページ（About Usのコンセプト文や実店舗写真、Size Guideの独自の採寸基準など）のテキスト原稿の提供。
  * 実在庫到着後の本番商品の写真データ提供。

## 4. 追加提案・推奨項目（運用トラブルを防ぐための事前準備）
設計書には存在しませんが、運用開始前にクライアントとすり合わせておくべき運用・保守のルールです。

* **本番用の商品撮影・画像処理ガイドラインの策定（玉記提案 / クライアント実行）**
  * サイトのレイアウト崩れや表示遅延を防ぐため、写真の縦横比（例：3:4で統一）、背景の色味、ファイル形式（WebP等）をルール化する。
* **本番商品の登録フロー・業務分担の明確化（玉記提案）**
  * オープン後、新商品が入荷した際の登録作業（マスタへの情報入力、ShopifyへのCSVインポート、画像アップロード、在庫数の入力）を「誰が」「どのように」行うかのオペレーションを構築する。
* **カスタマーサポート（CS）の対応フロー整備（クライアント実行）**
  * 問い合わせフォームの受信先メールアドレスの設定と、顧客からの返品依頼等が来た際の返信フォーマット・対応ルールを事前に決定する。
* **スタッフアカウントの権限管理（玉記提案 / クライアント実行）**
  * クライアント側のスタッフがShopify管理画面を操作する際、誤ってシステムを壊さないよう、権限範囲（売上データは見せるがテーマ編集は不可にする等）の設計を行う。
