# fullgram Portal Site メール通知＋Google Drive 自動保存のセットアップ

データ登録（ブランド・タグ・コレクション）の追加・削除・JSON出力時に、指定したメールアドレスへ通知を送り、Google Drive の共有フォルダに JSON ファイルを自動保存するための設定手順です。

## 1. Google Apps Script プロジェクトを作成

1. ブラウザで https://script.google.com/ を開きます
2. 「新しいプロジェクト」をクリックします
3. プロジェクト名を `fullgram-portal-mail-api` などに変更します（任意）

## 2. コードを貼り付ける

1. デフォルトで表示されている `コード.gs` の中身をすべて削除します
2. 同じフォルダにある `mail-api.gs` の内容をコピーします
3. `コード.gs` に貼り付けて保存します（Ctrl+S / Cmd+S）

## 3. Web アプリとしてデプロイ

1. 右上の「デプロイ」→「新しいデプロイ」をクリックします
2. 歯車アイコンの「種類の選択」→「Webアプリ」を選択します
3. 以下を設定します
   - 説明: `fullgram portal mail api`
   - Webアプリを実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**
4. 「デプロイ」をクリックします
5. 初回のみ Google の承認画面が表示されます。指示に従って許可してください
   - **Drive へのアクセス権限** も追加で求められることがあります。これは Google Drive への自動保存に必要な権限です
6. デプロイ完了後に表示される **WebアプリのURL** をコピーします

## 4. アプリに Web App URL を設定

1. fullgram Portal Site にログインします
2. 「タスク」→「データ登録」→「通知設定」を開きます
3. 「Google Apps Script URL」欄に、手順3でコピーした URL を貼り付けます
4. 「保存」をクリックします

## 5. 動作確認

1. 「ブランド管理」などのデータ登録画面を開きます
2. テスト用の項目を追加してみます
3. 指定したメールアドレスに通知メールが届くことを確認します

## トークンについて

`mail-api.gs` 内の `GAS_TOKEN` は、アプリと Google Apps Script の通信を保護するための文字列です。必要に応じて変更できますが、アプリ側の設定も同じ値にする必要があります。

- デフォルト: `fullgram-portal-token-2026`

## トラブルシューティング

- **メールが届かない / Google Drive に保存されない**
  - Web App URL が正しくコピーされているか確認してください
  - Google Apps Script のデプロイ時に「アクセスできるユーザー」が「全員」になっているか確認してください
  - スクリプトの実行権限の承認が完了しているか確認してください
  - Google Drive 共有フォルダに、GAS をデプロイした Google アカウントが **編集権限** で追加されているか確認してください
- **Google Drive 自動保存が失敗する**
  - GAS 内の `DRIVE_FOLDER_ID` が正しいフォルダ ID になっているか確認してください
  - GAS プロジェクトに Drive API の権限が承認されているか確認してください
- **CORS エラーが表示される**
  - Web App URL が `https://script.google.com/macros/s/.../exec` となっているか確認してください
